## Task HF2: CSP + security headers middleware

**Goal:** SPA отдаётся с жёсткими security-headers; `/api/*` и `/health` тоже.

**Files:**

- Create: `apps/server/src/middleware/security-headers.ts`
- Modify: `apps/server/src/server-bootstrap.ts` — подключить middleware до route handlers
- Create: `apps/server/tests/middleware/security-headers.test.ts`

**Implementation notes:**

- Headers:
  - `Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-eval' blob:; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self' data:; connect-src 'self'; worker-src 'self' blob:; frame-ancestors 'none'; base-uri 'self'; form-action 'self'` — `unsafe-eval` + `blob:` нужны Monaco Editor (Web Workers); `unsafe-inline` в styles — Tailwind JIT через `<style>` теги.
  - `X-Frame-Options: DENY`
  - `X-Content-Type-Options: nosniff`
  - `Referrer-Policy: no-referrer`
  - `Strict-Transport-Security: max-age=31536000; includeSubDomains` — только когда приходит HTTPS запрос (ловим по `x-forwarded-proto` если trusted-proxy, иначе по socket)
  - `Permissions-Policy: camera=(), microphone=(), geolocation=()`
- В dev-режиме (`NODE_ENV !== 'production'` ИЛИ `MIHARBOR_DEV=1`) CSP не включать или разрешить `ws:` для Vite HMR — чтобы не сломать `bun run dev`
- Reuse `TrustProxyEvaluator` из auth для HSTS-детекции (invariant: не доверять `x-forwarded-proto` от неизвестного proxy)

**Acceptance Criteria:**

- [ ] Middleware устанавливает все 6 headers на любой ответ, включая 404/static-SPA/HTML/JSON
- [ ] HSTS только когда запрос реально через HTTPS (socket.encrypted или trusted x-forwarded-proto=https)
- [ ] В dev-режиме CSP опущен (настраиваемо через `MIHARBOR_CSP_DISABLED=1` env)
- [ ] Тест покрывает: все headers присутствуют в prod; HSTS условный; dev-mode; trusted-proxy gate
- [ ] Сайт всё ещё загружается — Monaco + Tailwind не сломаны (ручная проверка оставляем под dogfood)

**Verify:** `bun test apps/server/tests/middleware/security-headers.test.ts` — зелёный. `bun run --filter='*' typecheck` — чисто.

## Task HF3: Unreachable linter — от O(n²) к линейному на практике

**Goal:** `detectUnreachable()` на 2000 правилах — <50 ms (против ~400 ms сейчас).

**Files:**

- Modify: `packages/shared/src/linter/unreachable.ts`
- Create: `packages/shared/src/linter/unreachable.bench.ts`
- Modify: `packages/shared/tests/linter/unreachable.test.ts` — добавить перф-кейс

**Implementation notes:**

- **Подход:** индексирование по `rule.kind` + `type` + построение reverse-suffix-index для DOMAIN-SUFFIX:
  - `exactIndex: Map<jsonKey, firstIndex>` — O(1) duplicate lookup
  - `suffixTrieByType: Map<type, Trie>` — при добавлении правила в trie, проверяем: покрывает ли уже существующий префикс current.value (обратная строка для suffix-match). Для `DOMAIN-SUFFIX,com` → в trie кладём `moc`. Для `DOMAIN-SUFFIX,example.com` — сначала ищем префикс `moc.` в trie (покрывает ли его `com` в trie).
  - DOMAIN-KEYWORD: не суффикс, а substring — trie тут не помогает. Оставляем O(n²) но **только для DOMAIN-KEYWORD правил**, которых обычно <50. Документируем.
  - MATCH catch-all: как только нашли — все последующие помечаются и break, это уже O(n).
  - IP-CIDR: TODO (как в текущей реализации), не оптимизируем.
- Поведение (output) идентичное старому — Issues с теми же `code`, `level`, `path`, `params`.
- Добавить `benchmark` тест который проверяет что 2000 правил обрабатываются за <50 ms (CI-tolerant: <200 ms).

**Acceptance Criteria:**

- [ ] Все существующие тесты `unreachable.test.ts` проходят без изменений output
- [ ] Новый перф-тест: 2000 правил (1000 DOMAIN-SUFFIX + 500 DOMAIN-KEYWORD + 500 IP-CIDR) — <200 ms
- [ ] Edge cases: trie не ломается на пустых значениях, Unicode, очень длинных доменах
- [ ] Бенчмарк-файл запускается отдельно через `bun run packages/shared/src/linter/unreachable.bench.ts` и печатает human-readable тайминги

**Verify:** `bun test packages/shared/tests/linter/unreachable.test.ts` + `bun run packages/shared/src/linter/unreachable.bench.ts`.

## Task HF4: Rate-limiter — disk-backed persistence

**Goal:** Lockout не сбрасывается на рестарт контейнера.

**Files:**

- Modify: `apps/server/src/auth/rate-limit.ts`
- Create: `apps/server/src/auth/rate-limit-store.ts` — persistence abstraction (load/save)
- Modify: `apps/server/src/auth/index.ts` — wire persistence
- Create: `apps/server/tests/auth/rate-limit-store.test.ts`
- Modify: `apps/server/tests/auth/rate-limit.test.ts` — добавить restart-сценарий

**Implementation notes:**

- **Storage:** JSON-файл `${MIHARBOR_DATA_DIR}/rate-limit.state.json`, атомарная запись через `tmp + rename` (можно переюзать хелпер из `transport/localfs.ts` если есть).
- **Debounce:** не писать на каждый fail — debounce 1 s. Но `process.on('SIGTERM'/'SIGINT')` flush немедленно.
- **Load:** при создании limiter-а в bootstrap — читает файл, парсит, отбрасывает entries с `lockedUntil < now` И expired fail-windows (консервативно: оставляем только то, что ещё актуально).
- **Graceful degrade:** если `MIHARBOR_DATA_DIR` не задан (тесты, dev) — persistence отключена, работает как раньше (pure in-memory).
- **Concurrent safety:** файл пишется только из master-процесса Miharbor (single-replica assumption). Документируем в коде.
- Обратная совместимость API: `createRateLimiter(opts)` не меняется; добавляется `createPersistentRateLimiter({...opts, storePath})` поверх него, либо через новый опцион `opts.store: Store`.

**Acceptance Criteria:**

- [ ] Новые тесты `rate-limit-store.test.ts`: save→load round-trip; expired entries отбрасываются; атомарная запись (mock fs с `rename` проверить).
- [ ] Тест restart-scenario: имитируем `createRateLimiter → 5 fails → save → dispose → reload → check() locked: true`.
- [ ] Существующие 30+ тестов в `rate-limit.test.ts` все зелёные без persistence (backwards-compat).
- [ ] `auth/index.ts` подхватывает persistence когда `MIHARBOR_DATA_DIR` задан.
- [ ] Secrets check: файл не содержит никакой auth-info кроме IP + счётчиков. IP нужен — это фича. Документируем retention: файл обрезается по `lockoutMs + failWindowMs` (auto-prune при load).

**Verify:** `bun test apps/server/tests/auth/` — все зелёные. `bun run --filter='*' typecheck` — чисто.

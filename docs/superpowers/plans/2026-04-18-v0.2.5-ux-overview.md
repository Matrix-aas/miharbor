# v0.2.5 UX — overview plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the three UX fixes specified in `docs/superpowers/specs/2026-04-18-geo-catalog-pending-diff-wg-unmask-design.md` — GEO-catalog combobox, pending-changes viewer with reset, and fix for `$MIHARBOR_VAULT:<uuid>` leaking into WG `public-key` field.

**Architecture:** Three orthogonal features delivered as three sequential PR-sized stages. Section 3 lands first (unblocks users), Section 2 second (small diff endpoint + modal), Section 1 last (parser + cache + combobox). Server-side stays in Elysia + yaml@2 + bun:test; client-side stays in Vue 3 + Pinia + vitest + diff2html.

**Tech Stack:**

- Server: TypeScript, Bun, Elysia, yaml@2, diff@9 (already shipped), bun:test.
- Client: Vue 3.5, Pinia, diff2html@3 (already shipped), vitest, @vue/test-utils.
- The `diff` package stays server-only — the plans add a `GET /api/config/draft/diff` endpoint rather than pulling `diff` into the web bundle.

---

## Execution order

Each section has its own plan file. Execute sequentially — later sections don't block earlier ones, but reviewing between stages keeps PRs small.

1. **[Section 3 — WG public-key unmask](2026-04-18-section-3-wg-public-key-unmask.md)** — 5 tasks (1-4 plus a regression test for legacy-snapshot rollback).

   Fixes the user-blocking bug where `$MIHARBOR_VAULT:<uuid>` appears in the WG public-key input and fails validation. Changes the vault `isSecretKey` predicate, adds `vault.resolveMany()`, migrates legacy drafts on read, and asserts legacy snapshots still unmask on rollback.

2. **[Section 2 — Pending-changes dialog](2026-04-18-section-2-pending-changes-dialog.md)** — 4 tasks.

   New `GET /api/config/draft/diff` server endpoint producing a masked-vs-masked patch via the existing `unifiedDiff` helper. New `PendingChangesDialog.vue` modal with diff2html render + destructive "Reset all" button. Header badge becomes clickable; the misleading "1 изменение" label is retired.

3. **[Section 1 — GEO catalog](2026-04-18-section-1-geo-catalog.md)** — 7 tasks.

   Minimal protobuf scanner for mihomo `.dat` files, URL resolution against `profile.geox-url.*` with MetaCubeX defaults, 24h in-memory cache, new `GET /api/catalog/geo`, new `GeoCatalogCombobox.vue` plugged into `RuleEditor.vue` for `GEOSITE` / `GEOIP` / `SRC-GEOIP` types.

## Global invariants

- **Tests live next to features.** Server tests go under `apps/server/tests/…`, web tests under `apps/web/tests/…`. `bun test` for server, `bun x vitest` for web. Both must pass before commit.
- **No new client-side dependencies.** `diff2html` is already shipped; `diff` stays server-only. Don't add `diff`, `protobufjs`, or similar to `apps/web/package.json`.
- **Coverage target ≥ 89.5 %.** Baseline is 89.98 %. Sections add meaningful lines — tests must cover new branches (vault-resolve-missing, first-fetch-failed, empty draft, etc.).
- **Per-section commit.** One commit per task; one merge per section. Do NOT bundle sections.
- **i18n keys always paired.** Every added key in `en.json` must also land in `ru.json` on the same task.
